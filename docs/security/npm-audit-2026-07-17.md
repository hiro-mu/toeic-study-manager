# npm audit 対応レポート (2026-07-17)

## 1. 対象範囲と方針

- ローカル開発環境で実行し、CI方針（.github/workflows/security-audit.yml）と整合させた。
- 今回の更新方針は、minor/patch を優先し、明示的な合意がない限りメジャー更新は行わない。
- 目的は、アプリの挙動安定性を維持しながら high/moderate 脆弱性を削減すること。

- node -v

- npm audit --audit-level=high --json > audit-high.json
- npm audit

- npm install next@15.5.20 eslint-config-next@15.5.20
- npm audit fix
- npm audit --json > audit-final.json
- npm run lint
- npm test -- --runInBand
- npm run build

## 3. 適用した変更

1. package.json から直接依存の git@0.1.5 を削除。
2. next を ^15.5.20、eslint-config-next を ^15.5.20 に更新。
3. 破壊的変更を伴わない npm audit fix を適用し、修正可能な間接依存の問題を解消。
4. 上記に合わせて package-lock.json を更新。

## 5. 残存脆弱性（詳細）

### A) firebase-tools 系列（合算で high 2 + moderate 4 相当の影響）

- 影響パッケージ: firebase-tools（直接 dev 依存）、@google-cloud/pubsub、@opentelemetry/core、gaxios、tar、uuid
- 依存チェーン:
  - firebase-tools -> @google-cloud/pubsub -> @opentelemetry/core
  - firebase-tools -> tar
  - firebase-tools -> gaxios -> uuid
- 残存理由:
  - npm audit 上の修正先が firebase-tools@15.24.0 であり、現行 14.x からの SemVer メジャー更新が必要。
- このプロジェクトでの到達性:
  - 主に開発用途（Firebase エミュレータ/ツール実行）で利用。
  - 本番向け Next.js アプリバンドルには firebase-tools は含まれない。
- 代表的なアドバイザリ:
  - GHSA-34x7-hfp2-rc4v（tar のパストラバーサル）
  - GHSA-8qq5-rm4j-mr97（tar の上書き/シンボリックリンク汚染）
  - GHSA-83g3-92jg-28cx（tar のハードリンクターゲット逸脱）
  - GHSA-qffp-2rhf-9h96（tar のドライブ相対パストラバーサル）
  - GHSA-9ppj-qmqm-q256（tar のシンボリックリンク経由パストラバーサル）
  - GHSA-r6q2-hw4h-h46w（APFS 上の tar レースコンディション）
  - GHSA-vmf3-w455-68vh（tar パーサ解釈差異）
  - GHSA-8988-4f7v-96qf（@opentelemetry/core のメモリ確保問題）
  - GHSA-w5hq-g745-h8pq（uuid の境界チェック不足）
- 暫定回避策:
  - firebase-tools の実行を、信頼できるローカル環境と CI に限定する。
  - ツール実行経路で信頼できないアーカイブを展開しない。
  - CI ランナーおよび開発端末のパッチ適用を維持する。
- 恒久対応案:
  - firebase-tools 15.x への移行トラックを別途実施し、互換性検証を行う。
  - 移行後に npm audit とエミュレータ動作確認を再実行する。

### B) next -> postcss の moderate 脆弱性

- 影響パッケージ: next（直接依存）、next 内包の postcss
- 依存チェーン: next -> postcss@8.4.31
- アドバイザリ: GHSA-qx2v-qp2m-jg93（PostCSS の </style> 非エスケープに起因する XSS）
- 残存理由:
  - 現在の依存解決では、next 配下の postcss が脆弱範囲として監査結果に残る。
  - npm audit の提案修正先に不整合（next 9.3.3 提示）があり、現行系列とメタデータ範囲の不一致が示唆される。
- このプロジェクトでの到達性:
  - 主に、信頼できない CSS 文字列を style コンテキストへ出力するケースで影響。
  - 本リポジトリはフレームワーク管理のビルド/実行経路が中心で、現時点で直接的な未信頼 CSS stringify 経路は確認していない。
- 暫定回避策:
  - ユーザー入力由来の生 CSS を style/script コンテキストへ注入しない。
  - UI 層で CSP と出力エスケープの運用を継続する。
- 恒久対応案:
  - next 側で内包 PostCSS が安全版へ更新される上流リリースを追跡する。
  - 修正版が確認でき次第、next を速やかに更新する。
